# Regions

You choose a region when you launch an instance, and the region sets where your
app runs. Amezmo picks a default in North America, and you can change it on the
launch screen.

Some regions don't appear for everyone. A region can be limited, in beta, or
reserved for a specific team, so you won't see it in the list unless it's
available to you. To launch in a region you don't see, contact support.

An instance you [share](sharing.md) with must run in the same region, so plan
shared instances into one region from the start.
